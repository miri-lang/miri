# Parser

The `parser` module is responsible for the second phase of compilation: **Converting a stream of tokens into an Abstract Syntax Tree (AST).**

## Overview

The Parser consumes the `TokenStream` generated by the Lexer and applies the grammatical rules of the Miri language to construct a hierarchical representation of the program. If the sequence of tokens does not form a valid Miri program, the parser reports syntax errors.

## Architecture

-   **Recursive Descent**: The parser is implemented as a hand-written recursive descent parser. Each major grammatical construct (e.g., `program`, `statement`, `expression`) has a corresponding parsing function.
-   **Pratt Parsing (Expressions)**: For expressions, the parser employs Pratt parsing (top-down operator precedence parsing). This elegantly handles operator precedence and associativity (e.g., ensuring `1 + 2 * 3` is parsed as `1 + (2 * 3)`).
-   **Error Recovery**: The parser includes mechanisms to synchronize and recover from local syntax errors. Instead of failing on the first unexpected token, it attempts to skip ahead to a known synchronization point (like a newline or statement boundary) so it can report multiple errors per pass.
-   **Indentation Awareness**: Miri relies on significant whitespace. The parser correctly interprets `Indent` and `Dedent` tokens to define the boundaries of blocks (such as function bodies or loop bodies), eliminating the need for mandatory curly braces.

## Design Principles

1.  **Predictive Parsing**: The parser looks ahead gracefully to determine which grammatical rule to apply, minimizing deep backtracking.
2.  **Noun-based Function Naming**: In accordance with the compiler's architectural guidelines, functions that parse and construct specific AST nodes are named as nouns (e.g., `identifier()`, `binary_expression()`) rather than verbs (`parse_identifier()`).
3.  **Precise Spans**: The parser combines the spans of the individual tokens it consumes to generate an accurate overarching `Span` for every AST node it creates.
